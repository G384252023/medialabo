    プロジェクト課題では、八王子で食べたいジャンルを選択るとそれに対応したお店を紹介してくれるページを作りました。 この課題では検索ボタンや選択肢の設定といったイベント処理、店舗名や住所といった情報を出すところまではすぐできましたが、2回目以降を押した時に前の値を消したり、 最初から何も書いていない箱が置いてあったのでそれらを消すのに苦労しました。また、最初に プログラムができて、情報が出た時に全てが同じ箱に入っていたので、それを分けるためのCSSとjsに苦戦しました。

    数当てゲーム

    数当てゲームはjavaScript,DOM,イベント処理の3つの要素を使ってランダムに設定された数字を 3回の質問で当てるゲームです。入力した値が答えより大きければ「もっと大きいですよ」と表示され、小さければ「もっと小さいですよ」 と表示され、回答が当たっていれば「正解です。おめでとう」と表示されるプログラムを組みました。

    第3回ではkazuate.htmlとkazuate.jsを作成し、数値を固定した状態でコンソールに表示するプログラムを組みました

    第4回ではDOMを使って、数当てゲームのページ改良をしました。また、yosoの値をコンソール上ではなく、HTML上に 出すようにしました。

    第5回ではイベント処理を利用し、テキストボックスや回答ボタンを設定しました。テキストボックスに入力した値を用いて、 HTML上に表示するのに苦戦しました。

    各授業回の学習事項

    第1回 WWW と HTML

    今回はHTMLについて学びました。HTML要素とはHTMLが要素から作られるものであり、 <p>や<strong>などといったタグを使って文章を作り、基本となるサイトを作れるようになった。 また、hrefを使ってリンク先のURLを入れることで、他のサイトに飛ばすこともできるようになった。

    演習の成果:演習1-4: HTMLのさまざまな要素

    第2回 CSS による Web ページの装飾

    今回は、CSSについて学びました。 CSSはWebページの装飾のために使うためのもので、CSSによって見た目（文字の太さや大きさ、色など）を変更できるということを学んだ。 HTMLで使うときはheadの中にrelとhrefを使って入れる。 どのような色使いや見た目にすれば見やすくなるかということに苦戦しました。

    演習の成果:練習問題2-3ウェブブラウザ説明ページの装飾

    第3回 JavaScript の基本

    今回はJavaScriptを使ったプログラムついて学びました。 java言語とは似ているようでintやletなど違う点がかなりあるため注意したい。 JavaScriptではjavaと違ってコンパイルの必要がなく、1行ずつ読み取ってくれるため、コンパイルの必要がない。

    演習の成果:練習問題3-6オブジェクトの参照

    第4回 DOM と JavaScript

    今回はDOMについて学びました。DOMはjavaScriptから文書の内容や構造を検索などをして文章や要素の追加や削除を行う。 仮のサイトからコンソールを使ってエラーを吐かないか試しながらプログラムを組むことで、エラーを吐いた時にわかりやすくなる。

    演習の成果:練習問題4-2~4-5説明文の追加、写真表を作る、箇条書きの削除、箇条書きの追加

    第5回 イベント処理

    今回はイベント処理について学びました。イベント処理ではWebページ上で起こった動作や出来事に対応した処理を 組み込むためのもので、マウスをクリックしたり、キーボードを押したりなど様々な処理に反応してイベントを処理する。

    演習の成果:練習問題5-3イベントを選択する

    • 工夫したところ

      • 自分や他の人が使っていて、目が痛くならないようにだったり、わかりやすい、見やすく使うためにはどのようにするべきなのかを考えて、色や配置を考えた。 特に色の設定をする際、濃い色を多用してしまうとその周辺の文字が見づらくなってしまったり、見やすいような色を使うように心がけると 常に同じような色になってしまうので自分の好きで見やすい色を使いつつ、同じ色でくどくならないように設定しました。

    • 苦労したところ

      • 特に苦労したのはDOMの操作とjavaScriptの作成です。

      DOMの操作をしている時は、演習や課題をやっている時に何度HTMLの方をそのまま編集したら終わるなぁ...と考えたかわかりません。

      javaScriptの作成ではやはりjavaとの微妙な違いに困惑しました。いつも通りVScode上に書いてしまったり、思った通りの行動をしてくれなかったり、 実行した際にエラーが頻発してしまいました。その度に間違えたところを確認して、javaScriptを編集してエラーを修正してを繰り返していました。

      ただDOMもjavaScriptも、コンソールを使って書くようにしたことで少し苦手意識は無くなりました。

    • 強調したところ

      • 工夫したところでも書きましたが、CSSの設定で見ていてしんどかったりくどくならないような色や配置になるように心がけました。

    • 感想

      • 今回の情報メディア実験「Webコンテンツ作成」ではHTML,CSS,javaScriptを使って、Webページがどのようにして作られているか、自分でWebページを作るやり方についてを学んできました。 また、今回の授業によって別のWebページを見たときにこれはこうやってつくってるのか。このフォントどうやって出しているんだろう。など様々な視点からWebページを見れるようになりました。 この授業が始まった時、僕はHTMLやjavaScript、CSSについてほとんど何も知らなかったため、毎回の講義中で言われていることについていくだけで精一杯で、途中で行う演習で だいぶギリギリの状態になっていました。なので毎回のプロジェクト課題まで手が回らず、第6回ではプロジェクト課題にほとんど手がついておらず、大変でしたが、普段使っているサイトが どのようにして作られているかを学べたことはとても有意義なものとなりました。 そして、プロジェクト課題や数当てゲームの課題をやっている時はわからないことも多く、アシスタントの方々や教授に教えてもらいながらではありますが、なんとか終わらせることができました。 javaを初めて学習したときも思いましたが、新しいプログラミング言語を一つ知るたびにできることが山のように増えていくのが とても楽しかったです。今回の授業で学んだ経験を活かして今後社会に出たときに自分でWebページを作っていきたいです。
